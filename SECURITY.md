# Security Policy

## Supported Versions

| Version | Supported         |
| ------- | ----------------- |
| 1.x     | ✅ Active support |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do not** open a public GitHub issue.
2. Email the maintainers with a detailed description of the vulnerability.
3. Include steps to reproduce the issue, if possible.
4. Allow reasonable time for the issue to be resolved before public disclosure.

## Embargo Policy

- Security fixes will be developed and tested privately.
- A patch release will be published as soon as a fix is verified.
- Public disclosure will occur after the fix is released.
